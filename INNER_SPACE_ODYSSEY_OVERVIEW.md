# 内太空漫游 (Inner Space Odyssey) - 项目白皮书与技术全览

**文档受众：开发人员、新接入的 AI Agent**
**核心作用：提供项目的系统架构、业务逻辑、美学定位与详细功能说明的全局 Context。**

## 🚀 1. 项目背景与宏观定位 (Project Vision & Positioning)

### 1.1 破局痛点 (Pain Points)

在现代信息社会，人们对于探讨自己内心深处的真实诉求（尤其是两性倾向、深层防御机制、甚至隐匿的癖好）往往感到难以启齿、充满社交戒备感。传统的性格测试表单极其枯燥且容易让人产生“被评判”的抵触情绪。

### 1.2 核心理念与美学 (Core Philosophy & Cyber-Aesthetics)

**《内太空漫游》** 是一款主打“强科幻叙事互动 + 大模型心理体检 + A2A (AI-to-AI) 盲聊试探”的硬核心理与两性治愈 Web 应用程序。

- **科幻作为“保护色”**：我们利用极端冷峻、硬核的“深空探测”、“星舰航行”、“物理机能”等宏大会切断现实联系的隐喻，来重新包装那些敏感的心理学名词。通过将测试**“去评判化、去性化”**，让用户在一个拥有绝对安全感和史诗感的深空医学语境下卸下所有防备。
- **赛博复古极客美学 (Retro Cyber-Geek)**：全站 UI 摒弃常见的小清新或白板风格，手工手写大量 CSS，采用**全局黑底、荧光青绿 (Cyan) 与深紫 (Purple) 的赛博配色**。深度还原 CRT 屏幕扫描线、终端打字机效果、噪点覆盖与全息发光边缘，营造电影级的沉浸心流。

## 🛠 2. 核心功能链路详解 (Core Feature Pipelines)

用户在整个应用中的生命周期分为以下四个完整阶段（按页面路由演进）：

### 📡 Phase 1: 身份互联层 (OAuth & Auth) —— `/login` & `SecondMe 整合`

- **去表单化登陆**：应用不支持传统账密注册，强制使用自建的 **SecondMe 开放平台 OAuth2** 标准授权登入。
- **一键潜航机制**：一键授权后，应用拉取用户的 SecondMe 数字分身名片库（如：ID、简介、深层价值观“软记忆/SoftMemory”、兴趣爱好“Shades”），并持久化到本地 Supabase/Prisma 数据库中的 `User` 表。不仅简化注册，还为后续 AI 大模型提供极其丰富的背景知识输入。

### 🪐 Phase 2: 数据采集层：深空探勘剧情 (Scenario Interactive Test) —— `/scenario`

- **沉浸式绝境抉择**：摒弃传统滑动条输入。用户将代入 5 个不同场景的超沉浸科幻绝境（例如：面对空间站爆炸，是牺牲自己关闭辐射，还是带着资料独活？在低温睡眠舱中，希望是恒温还是冰冻？）。
- **参数隐性重度提纯**：用户的每个选择，其实际背后映射了四大底层核心维度参数（取值 0-100）：
  1. **防线韧性 (Defense Level)**：心理对外界的防御值与安全感渴求度。
  2. **环境偏温带 (Temp Preference)**：热情如火到冰冷理智的体感偏好分布（从“熔毁”到“极寒”）。
  3. **共振频率 (Rhythm Perception)**：生物钟与感官的刺激频率偏好。
  4. **隐密阈值 (Hidden Need)**：最后一步，允许用户在一个复古文本终端手打一段自己完全没有保留的“隐秘诉求”或“怪癖”。
- **全网共鸣统计锚点 (Global Choice Statistics)**：每次抉择后，屏幕正中会触发 **打字机动态特效**，实时调取数据库并弹回类似：“在目前启航的 23,010 艘星舰中，只有 4.2% 的舰长作出了同样的抉择。”这种宏大的数据披露，极具窥探欲，能够瞬间触发用户的自豪感或“我不孤单”的情感慰藉（参考《底特律：变人》的通关统计仪）。

### 🧬 Phase 3: 算力揭晓层：双线星舰蓝图结果页 (Blueprint Dashboard) —— `/blueprint` & `/blueprint-solo`

根据系统是否为该参数匹配到了合适的实体，路由会分流至“单人沉浸”或“双人匹配”版本，但核心揭底模块一致：

- **极坐标多维雷达网格 (Echarts Radar)**：用雷达图谱将提纯好的四维护生物学参数可视化。
- **大模型临床医学剖析 (`/api/blueprint-analysis`)**：这是后端强劲的 AI 引擎核心。前端将包含隐秘诉求的参数发送给大模型（MiniMax API），要求其扮演“深空情感科室主任”：
  1. **医生诊断**：分析心理疲劳与感官压力。
  2. **工程师架构**：为该用户量身定制一台物理理疗设备及材质（如：_推荐TPE隔离硅胶涂层，38°C安全恒温_）。
  3. **隐患排雷**：AI 会抓取用户手写的秘密怪癖，用毫无偏见且充满温柔庇护感的话语给出回应。
- **SSR 级「专属星舰档案」海报打印站 (Poster Generator)**：
  - **动态 AI 称号授予**：根据数值的畸变程度（越畸形/怪异/罕见，稀有度越高），AI 自动下发 N、R、SR 乃至极高贵的 **SSR** 级治愈系史诗称号（如：_【SSR】深空凛冬的守望者_，_【SR】时间停滞的安息地_）。
  - **社交货币裂变**：利用 `html-to-image`，在前端把雷达图谱、AI诊断流、条形码生成长图，供用户截图保存并发往小红书/朋友圈装X。

### 🤖 Phase 4: A2A (AI-to-AI) 矩阵盲聊系统 —— （在蓝图页面底部触发）

这是本系统最具实验性、最能规避人类直男/社恐交互障碍的创新环节。
由系统生成属于玩家本人的 AI 分身，去替玩家进行“思想体检”或者“赛博相亲”。

- **【模式 A：单机内心互搏】(SoloInnerChat)**
  系统启动两个不同向位的内置特工模型：一个代表玩家的“绝对现实理智”，一个代表“本能的纯粹欲望”。由前端在屏幕中上演激烈的对话辩论。
  _效果_：宛若脑内剧场，让玩家以上帝视角观看自己隐秘想法与理智的搏斗。
- **【模式 B：双星博弈与知乎截获】(BlueprintChat)**
  系统在数据库中探测到高契合度真实玩家（基于欧几里得距离与余弦相似度算法 `matchUtils.ts`），并提取对方数据。
  **核心机制 - “知乎原生共鸣截获”**：
  为避免尬聊，系统会自动请求 `/api/zhihu/search`，利用用户偏好参数在知乎实时抓取类似“如何看待亲密关系中的窒息感？”这类硬核高质量情感研讨贴。以此为抓手，由代表“我”的AI去询问对方的AI，对方AI再进行回应。
  _效果_：双边 AI 围绕硬核知乎议题进行长达 4-6 轮的极限拉扯与三观勘探，确认极度同频且安全后，才会暴露出各自的真实 SecondMe 主页链接。

## 💻 3. 工程与技术栈地图 (Engineering & Tech Stack)

### 前端生态 (Frontend Context)

- **核心框架**：Next.js 14+ (App Router)
- **UI & CSS**：TailwindCSS + 手写底层动画关键帧 (keyframes.css)。大量运用 backdrop-blur、mix-blend-mode、box-shadow 创造深空霓虹与 CRT 质感。
- **图表展示**：ECharts (`echarts-for-react`) 定制雷达图。
- **状态管理**：Zustand (`useAgentStore`) 承载贯穿全局的问卷积分流转。
- **流式对话**：通过 Fetch 与 SSE (Server-Sent Events) 手工实现与后端大模型的打字机流式响应组件。

### 后端与 API 层 (Backend API Routes Context)

所有 API 皆为 Next.js Route Handlers。核心文件在 `src/app/api/` 下：

- **`/api/agent-prefill`**：流式对话大模型核心枢纽（Minimax 接入点），负责操控两个对战的 AI 分身产生互相对话的历史队列和推理内容。
- **`/api/blueprint-analysis`**：用于非流式的一键生成深空医学报告、以及算出长图海报上的定制化 SSR 称号。
- **`/api/stats`**：直连 Prisma 数据库，返回真实的现存注册舰长数量，注入全局统计面板。
- **`/api/zhihu/search`**：负责抓取外界社区高质量议题的爬虫通道。
- **`/api/match-ai`**：匹配引擎，根据当前 payload 进行数据库全量遍历，寻找各项矩阵值相交并补充的实体。

### 数据库层 (Database Model - Prisma)

关键实体：

1. **User**: 包含了从 oauth 带回来的关联资料 JSON (shades, softMemory)，同时存储了其做完测试后入库的 defenseLevel、tempPreference 等指标。
2. **KnowledgeCard & ScenarioTip**: 保存了知乎/医学相关的静态配置与场景选择提示。

## 🌟 4. 全局核心总结 (Summary for Integration)

无论你在开发什么新功能，请务必保证：

1. **绝不允许破坏赛博科幻医学的沉浸感**。所有报错、提示语都要包装成如“信号丢失”、“舱门密封失效”等中二台词。
2. **永远包容用户的癖好**。在 AI 生成相关逻辑时，确保输出方向是治愈、接纳、去偏见的，严禁说教。
3. **优先保证移动端体验**。因为这东西重度依赖微信/小红书裂变等移动端社媒分享，DOM结构需极其弹性。
